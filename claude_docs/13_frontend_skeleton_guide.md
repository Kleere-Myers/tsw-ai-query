# Frontend Skeleton Guide (Angular)

This document defines frontend structure and behavior.

## Core Principles
- Typed contracts everywhere
- No schema knowledge in UI
- Backend is source of truth
- UX favors clarity over power

## Feature Areas
- Ask: natural language querying
- History: user query history
- Admin: governance configuration

## Ask UX
- State machine driven
- Clarification-first
- Results with explanations

## Admin UX
- Guardrail-driven CRUD
- Inline editing
- No free-form SQL

## Error Handling
Errors must be friendly and actionable.
